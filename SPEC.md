# 写真展 POS システム 仕様書

**バージョン：** v1.0  
**作成日：** 2026年6月1日  
**システム名：** 写真展 POS

---

## 1. システム概要

写真展の当日運用を支援する簡易POSシステム。  
Google Apps Script（GAS）をバックエンドとして、iPadなどのタブレットからブラウザで利用できるWebアプリ。  
Googleスプレッドシートにリアルタイムで売上データを記録する。

---

## 2. システム構成

```
┌─────────────────────────────────────────┐
│  iPad / スマートフォン / PC               │
│  ブラウザ（Safari / Chrome / Firefox）   │
│  ※ Googleアカウントへのログインが必要     │
└────────────────┬────────────────────────┘
                 │ HTTPS
┌────────────────▼────────────────────────┐
│  Google Apps Script（GAS）Webアプリ      │
│  ・index.html を配信                     │
│  ・google.script.run でAPI通信           │
│  ・スプレッドシートへの読み書き           │
└────────────────┬────────────────────────┘
                 │
┌────────────────▼────────────────────────┐
│  Google スプレッドシート                  │
│  ・売上明細シート                         │
│  ・商品マスターシート                     │
│  ・担当者マスターシート                   │
└─────────────────────────────────────────┘
```

## 4. 機能一覧

### 4-1. レジ機能

| 機能 | 内容 |
|------|------|
| 担当者選択 | レジ上部のチップボタンまたはヘッダーのモーダルから選択 |
| カテゴリ選択 | チケット／物販を切り替えて表示 |
| サブカテゴリ選択 | 物販選択時にサブカテゴリタブで絞り込み |
| 商品タップ | 商品ボタンをタップしてカートに追加 |
| 数量変更 | カート内の＋／－ボタンで数量変更 |
| 商品削除 | カート内の✕ボタンで削除 |
| 合計表示 | カート内容に応じてリアルタイム計算 |
| 支払い方法選択 | 現金 or PayPay |
| お預かり入力 | テンキーパッドで入力（現金選択時） |
| おつり計算 | お預かり金額入力後に自動計算・表示 |
| 精算 | 担当者・支払い方法・金額が揃ったら「精算する」ボタンが有効化 |
| 精算完了モーダル | 精算内容を確認後「次の会計へ」で次の会計へ |
| カートクリア | 「カートをクリア」ボタンでリセット |

### 4-2. 売上履歴機能

| 機能 | 内容 |
|------|------|
| 総売上表示 | 当日の総売上・件数 |
| 支払い別集計 | 現金・PayPay別の売上・件数 |
| 担当者別集計 | 担当者ごとの売上・件数 |
| 取引明細一覧 | 時刻・担当者・取引番号・商品内訳・金額・支払い方法・おつり |
| CSVダウンロード | 売上明細をCSV形式でダウンロード |
| 履歴クリア | 当日の履歴をクリア |

### 4-3. 商品マスター機能

| 機能 | 内容 |
|------|------|
| 商品一覧表示 | コード・商品名・単価・種別・サブカテゴリ |
| 商品追加 | 「＋追加」ボタンで行を追加 |
| 商品編集 | 各フィールドを直接編集 |
| 商品削除 | 「削除」ボタンで行を削除 |
| スプレッドシート保存 | 「💾 スプレッドシートに保存」でGASを通じて保存 |

### 4-4. 担当者マスター機能

| 機能 | 内容 |
|------|------|
| 担当者一覧表示 | 担当者名の一覧 |
| 担当者追加 | 「＋追加」ボタンで行を追加 |
| 担当者編集 | 名前を直接編集 |
| 担当者削除 | 「削除」ボタンで行を削除 |
| スプレッドシート保存 | 「💾 スプレッドシートに保存」でGASを通じて保存 |

### 4-5. 設定機能

| 機能 | 内容 |
|------|------|
| 接続テスト | GASとの通信確認 |
| キャッシュクリア | ローカルキャッシュをクリアしてスプレッドシートから再同期 |

| サブカテゴリ | アイコン |
|------------|---------|
| ノート | 📓 |
| ファイル | 📁 |
| ポストカード | 🗂 |
| マグネット | 🧲 |
| ポストイット | 📌 |

### 商品データ項目

| 項目 | 内容 |
|------|------|
| コード | 商品を一意に識別するコード（例：NT-001） |

| 単価 | 円（整数） |

## 6. スプレッドシート仕様

### シート構成

| シート名 | 用途 | 更新タイミング |
|---------|------|--------------|
| 売上明細 | 精算データの記録 | 精算のたびに自動追記 |
| 商品マスター | 商品一覧 | アプリから手動保存時 |
| 担当者マスター | 担当者一覧 | アプリから手動保存時 |

### 売上明細 カラム定義

| カラム | 内容 |
|--------|------|
| 日時 | 精算日時（YYYY/MM/DD HH:MM:SS） |
| 取引番号 | TX-0001形式の連番 |
| 担当者 | 選択した担当者名 |
| 商品名 | 商品名 |
| カテゴリ | チケット／物販 |
| サブカテゴリ | 物販のサブカテゴリ |
| 数量 | 購入数量 |
| 単価 | 商品単価（円） |
| 小計 | 単価×数量（円） |
| 合計 | 取引合計金額（円） |
| 支払方法 | 現金／PayPay |
| お預かり | 現金の場合のお預かり金額 |
| おつり | 現金の場合のおつり金額 |

## 7. GAS API仕様

### エンドポイント

GASのWebアプリURLに対して`google.script.run`で通信する。

### アクション一覧

| アクション | 内容 |
|-----------|------|
| `getMasters` | 商品マスター・担当者マスターを取得 |
| `saveMaster` | マスターデータをスプレッドシートに保存 |
| `addSales` | 売上明細をスプレッドシートに追記 |
| `ping` | 接続確認 |

---

## 8. データ管理仕様

| 項目 | 内容 |
|------|------|
| 起動時 | GASからマスターデータを自動読み込み |
| オフライン時 | ローカルキャッシュ（localStorage）を使用 |
| 売上データ | ローカルに保存しつつGASへ非同期送信 |
| GAS送信失敗時 | ローカルには保存済み（ログに警告） |

---

## 9. 支払い仕様

| 支払い方法 | 処理 |
|-----------|------|
| 現金 | テンキーでお預かり金額入力→おつり自動計算→精算 |
| PayPay | 支払い方法選択のみ→精算（金額入力不要） |

## 10. 運用フロー（当日）

```
開場前
  └── iPadでGASのURLを開く（Googleログイン済みであること）
  └── 担当者を選択
  └── 接続テストで動作確認

営業中
  └── チケット or 物販を選択
  └── 商品をタップしてカートに追加
  └── 支払い方法を選択
  └── 現金の場合はテンキーでお預かり金額を入力
  └── 「精算する」をタップ
  └── 精算完了モーダルを確認して「次の会計へ」

閉場後
  └── スプレッドシートで売上明細を確認
  └── 担当者別・支払い別の集計を確認
  └── 必要に応じてCSVダウンロード
```
